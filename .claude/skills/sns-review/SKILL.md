# /sns-review — SNS実績レビュー（伸びた型→翌週へ）

計測データから「どの柱・どの型が伸びたか」を分析し、翌週の `/sns-pack` に渡せる方針を出す。
過去に断念した「配信への理解を深めた上でのコンテンツ作成」を、このループで仕組み化する。

## 入力

次のいずれか。**どれも無ければ、iPhoneで数字を読み上げ/貼り付けしてもらう形式を案内して、貼り付けを待つ**:
1. `docs/sns/metrics/YYYY-MM.csv`（推奨。形式は `docs/sns/metrics/README.md`）
2. チャットへの数字貼り付け（雑な形式でよい。こちらで構造化する）
3. Notion「X投稿カレンダー」（`2c2b426bf123473894a98c1adcff2947`）に実績列があればそこから

## 手順

### 1. データ整形
- 貼り付け入力だった場合は、CSV形式に整形して `docs/sns/metrics/YYYY-MM.csv` に**追記**（既存行は消さない）
- 投稿は柱（A天気/B相談所リアル/C問いかけ/Dつくる日記/E告知/F中の人）に分類

### 2. 分析（機械的に・感想で済ませない）
- 柱ごとの中央値と最高値（imp・エンゲージ率・プロフィールクリック率）
- トップ3投稿の共通点（型・時間帯・書き出し・長さ・会話設計）
- ワースト3投稿の共通点
- プロフィールクリック→リンククリックの歩留まり（集客導線として機能しているか）
- 前週比の変化（データが2週以上あれば）

### 3. レポート出力
`docs/sns/reviews/YYYY-Wnn.md` に保存:
- 今週のサマリー（3行）
- 柱別の成績表
- 伸びた型（次週に厚くする）/ 沈んだ型（減らす・変える）
- **翌週の方針（/sns-pack にそのまま渡せる指示文の形で書く）**
- 計測できていない指標があれば「来週から取る数字」を1つだけ提案（欲張らない）

### 4. 反映
ユーザーが承認したら、翌週分の `/sns-pack` 実行時にこのレビューの方針を読み込んで生成する
（sns-pack 側の手順1で最新の `docs/sns/reviews/` を参照）。

## 判断の基準（Kinda固有）
- 「バズ」より「検討層に届いたか」（プロフィールクリック・リンククリック・フォロー転換を上位指標に）
- フォロワー数の増減で一喜一憂しない。**保存・返信・プロフ遷移**という深い行動を重く見る
- 柱F（中の人）は別枠計測（TODO の方針どおり）
- 数字が小さいうちは統計的に確定させない。「仮説→翌週試す」の言い方にとどめる

## 禁止事項
- 実績CSV・Notionの既存データの削除・改変（追記のみ)
- 数字を伴わない「なんとなく良かった」で方針を変えること
