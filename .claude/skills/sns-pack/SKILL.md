# /sns-pack — SNS投稿パック一括生成

1つのソースから **X（5〜7本）・Instagramカルーセル（1本）・note記事案（1本）** をまとめて生成し、
QA後に Notion 投稿カレンダーへ**下書き**として投入する。**SNSへの実投稿は絶対にしない**（投稿は人間）。

## 入力

引数またはユーザーから受け取る。足りなければ推測で埋めて冒頭に前提を明示（質問で止まらない）:
- **ソース**: `content/columns/<slug>.mdx` のslug / story id / 自由テーマ のいずれか
- **対象週**: 省略時は「今日から7日間」

## 手順

### 1. 文脈を読む（毎回・順に）
1. `CLAUDE.md` §1〜§5（憲法・トーン）— セッション冒頭で読了済みなら再読不要
2. `docs/sns/sns-handoff-for-claude-ai.md`（SNS運用の現在地）
3. ソース本体（コラムMDX / `src/lib/mock/stories.ts` の該当story / テーマ）
4. Notion「X投稿カレンダー」（database_id `2c2b426bf123473894a98c1adcff2947`）の対象週を確認し、
   既存予定と**日付・柱の重複を避ける**

⚠️ story をソースにする場合、`consent.sns === true` の story のみ使用可。false/未記録なら**その場で中断して理由を伝える**。

### 2. X投稿を生成（5〜7本）
5本柱ローテに沿って生成する:
- **A 天気の言葉**（詩的・気持ちのメタファー）/ **B 相談所リアル**（一次情報）/ **C 問いかけ**（会話設計）/
  **D つくる日記**（運営の等身大）/ **F 中の人**（クスッ・昼12時台）/ E 告知は必要時のみ

ルール（アルゴリズム対応・2026調査済み）:
- **外部リンクは本文に入れない**。リンクは「セルフリプ用」として別行で添える
- 同日連投しない（Author Diversity）。朝・昼・夜に分散した日時案を付ける
- 1本140字前後。問いかけ形は返信しやすい低ハードルの問いに
- 絵文字は最小限・区切り用途のみ。ハッシュタグはXでは0〜1個

### 3. Instagram を生成（1本）
- カルーセル5〜7枚の**各枚テキスト案**（1枚目=フック、最終枚=CTA「プロフィールのリンクから」）
- キャプション（リード2行＋本文＋ハッシュタグ10個前後：婚活系ビッグ＋ミドル＋Kinda固有）
- 画像はクレイ風トーン（`#F5EEE6`/`#D4A090`）の生成プロンプト案を添える（生成自体は任意）

### 4. note記事案を生成（1本）
- タイトル案3つ＋構成（見出し＋各節の要旨）＋リード文200字
- 本文まで書くかはユーザーに委ねる（パックには構成まで含める）

### 5. セルフQA（全文に対して）
- [ ] 焦らせ・比較表現なし（「早くしないと」「周りは」禁止）
- [ ] 「中立」を自称していない
- [ ] 「結婚・成婚」をゴールとして煽っていない（中核語彙=「好きな人」「ふたり」「過ごす日々」）
- [ ] 「異性」「男女」前提の表現なし
- [ ] bio・ブランドコピー的な文には婚活語なし（投稿本文はOK）
- [ ] 空虚な共感（「わかるよ」だけ）で終わる投稿がない
- [ ] 他人の物語の無断利用なし（story は consent.sns=true のみ）

### 6. パックをファイル保存
`docs/sns/packs/YYYY-MM-DD-<slug>.md` に全文保存（再利用・履歴資産）。
形式: 冒頭にソース・対象週・QA結果、以降 X→IG→note の順。

### 7. Notion へ下書き投入（要確認）
保存後、**投入内容の要約（本数・日時・柱）を提示してユーザーの承認を得てから**:
- X → 「X投稿カレンダー」DB（`2c2b426bf123473894a98c1adcff2947`）へ状態=下書きで追加
- IG → 「IG投稿カレンダー」DB（Notion検索で特定）へ同様に追加
既存行は**編集・削除しない**。追加のみ。

## 禁止事項
- X/Instagram/note への直接投稿・予約投稿（Notion下書きまでが役割）
- consent.sns の確認なしに story 素材を使うこと
- Notion の既存カレンダー行の変更・削除
