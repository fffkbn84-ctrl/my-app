## 11. Kinda voices 運用フロー（取材→記事化）

> Kinda voices はカウンセラー個人単位の取材記事セクション。
> 掲載初期の信頼担保・SEO・営業ツールの3役を兼ねる。
> このセクションは毎回の取材作業の標準手順として参照すること。

### 記事の基本設計

| 項目 | 内容 |
|---|---|
| 単位 | カウンセラー個人単位（1相談所1人から始める） |
| 推奨文字数 | 3,000〜4,500字 |
| 推奨画像 | 5〜8枚（取材写真） |
| 主要SEOキーワード | 相談所名＋カウンセラー名＋地域名をtitleに |
| 内部リンク | 記事末尾 → カウンセラー詳細ページ |
| CTA文言 | 「〇〇さんの話を、直接聞いてみる。」（緊急感・比較禁止） |

### 記事構成（8セクション・固定）

1. タイトル＋ヘッダー画像
2. リード文（200字以内・核心を先に）
3. プロフィールサマリー（箱組み）
4. この仕事を選んだ理由（原体験）
5. どんな人を担当してきたか
6. 相談所の空気感・働き方
7. 印象に残っているひとこと（引用ブロック・SNS拡散設計）
8. クロージング＋CTA → カウンセラー詳細ページへの内部リンク

### Kindaトーン（Kinda voices記事専用の注意）

- 励まさない・断言しない・現象描写にとどめる（Kinda note と同じ原則）
- 「婚活」「結婚」はSEO上本文で使ってOK。ただしゴールとして煽らない
- 「婚活しなくちゃ」より「誰かと過ごしたい」に寄り添う視点で書く
- 引用ブロックには「業界の常識をそっと外すような言葉」を選ぶ

### 取材→記事化ワークフロー（MacBook Neo環境）

```
① iPhoneのボイスメモで取材を録音

② AirDropでMacのデスクトップに転送

③ ターミナルで以下を実行
   whisper ~/Desktop/ファイル名.m4a --language ja --model medium
   （初回のみmediumモデルDL・約5分）

④ デスクトップに .txt ファイルが自動生成される

⑤ Claude.aiのチャットに貼り付けて
   「Kinda voicesの構成で記事にして」と指示

⑥ 完成原稿をMDXファイルとしてKinda voicesに追加
```

### Whisper環境（MacBook Neo・設定済み）

- Homebrew・ffmpeg・openai-whisper（Python 3.9）インストール済み
- PATH設定済み（~/.zprofile）
- 費用：完全無料（ローカル処理）
