## このセッションで確定した事項

### 1. 決済設計（確定・実装待ち）

**Stripe構成：**
- 使用機能：Stripe Payments（Payment Intents）+ Stripe Customers + Webhook + Refund API
- 支払い手段：**クレジットカード登録制のみ**（銀行振込は対応しない・例外なし）
- タイミング：予約成立時に即時前払い ¥5,000

**課金フロー：**
```
ユーザーが予約確定
    ↓
Stripeが相談所のカードに即時課金（¥5,000）
    ↓
決済成功のWebhookをKindaが受信
    ↓
ユーザーの連絡先・情報が相談所管理画面に自動開示
    ↓
面談実施
```

**返金ルール（05-30セッション確定分と合わせた完全版）：**

| ケース | 処理 |
|---|---|
| 24時間以内キャンセル（誰都合でも） | Refund API で即返金 |
| 24時間以降・ユーザー都合キャンセル | 返金なし・課金確定 |
| 24時間以降・相談所都合キャンセル → 別日了承 | 返金 → 再課金 |
| ユーザーが日程変更希望 → 別日了承 | 返金 → 再課金 |
| 上記以外の面談不成立 | 返金なし・課金確定 |

**クレカなし相談所の対応方針：**
- 対応しない。カード登録を契約条件とし、持っていない場合は契約不可として断る。

**Stripe手数料：**
- 3.6%のみ（Billing・Invoicing不使用のため最小コスト）

### 3. Kinda Voices 記事テンプレート（確定）

**概要：**
Greenの企業インタビュー形式をKindaに適用。カウンセラー個人単位の取材記事を Kinda voices に積み、カウンセラー詳細ページからもリンクするダブル使い設計。

**記事構成（8セクション）：**
1. タイトル＋ヘッダー画像（SEO：相談所名＋カウンセラー名をtitleに）
2. リード文200字以内（核心を先に・離脱防止）
3. プロフィールサマリー箱組み（地域名・得意ケース）
4. この仕事を選んだ理由（原体験・具体エピソード）
5. どんな人を担当してきたか（ロングテールSEO狙い）
6. 相談所の空気感・働き方（心理的ハードル低下）
7. 印象に残っているひとこと（SNS拡散設計・引用ブロック）
8. クロージング＋CTA（「〇〇さんの話を、直接聞いてみる。」→内部リンク）

**推奨スペック：**
- 文字数：3,000〜4,500字
- 画像：5〜8枚（取材写真）
- 主要SEOキーワード：相談所名＋カウンセラー名＋地域名
- 内部リンク先：カウンセラー詳細ページ

**Emma取材用 質問リスト20問：** handoff内に含まれているため省略。必要なら前チャット参照。

### 4. Whisperセットアップ（完了）

MacBook Neo（macOS）に以下のインストール済み：
- Homebrew（/opt/homebrew）
- ffmpeg
- openai-whisper（Python 3.9）
- PATH設定済み（~/.zprofile）

**取材後の使用コマンド（これだけ覚える）：**
```bash
whisper ~/Desktop/録音ファイル名.m4a --language ja --model medium
```

**取材→記事化ワークフロー：**
```
iPhoneボイスメモで録音
    ↓
AirDropでMacのデスクトップに転送
    ↓
ターミナルでwhisperコマンド実行
（初回のみmediumモデルDL・5分程度）
    ↓
デスクトップに.txtファイル自動生成
    ↓
Claude.aiに貼り付け
「Kinda voicesの構成で記事にして」
    ↓
完成原稿
```
