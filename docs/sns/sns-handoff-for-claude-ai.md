# Kinda SNS発信 引き継ぎ（Claude.ai 用ブリーフ）

> 正（憲法）は `CLAUDE.md`。本ファイルはその SNS 運用向け要約＋現在地。最終更新: 2026-06-09。

## 0. Kinda とは（30秒）
結婚相談所を「**カウンセラー個人の口コミ**」で選んでから予約できる、口コミ・レビュー型プラットフォーム（食べログの結婚相談所版）。本番稼働中：**https://kinda.jp**。
- 外部での自称：「**ユーザーファーストな口コミ・レビュー型プラットフォーム**」。⚠️「中立」とは言わない。
- メインターゲット：結婚相談所を探している**未入会者**＋関係を築いている最中のすべての人（LGBTQ+ 含む）。

## 1. SNS媒体方針
- **note ＋ Instagram を主軸**。**X は記事の引用を流す軽運用**。Threads/TikTok/BeReal は後回し（ひとり運用＝2媒体集中）。
- 2026年トレンドと整合：**バズより信頼・一次情報・盛らない**。Kinda のブランドはこの潮流のど真ん中なので意図的に武器化する。

## 2. ブランドの声（全 SNS 共通・厳守）
- **焦らせない・比較しない**（「早くしないと」「周りは」で追い詰めない。自分のペースを肯定）。
- **空虚な共感を禁止**（「わかるよ」で終わらせず実用価値を返す）。
- **LGBTQ+ 包摂**（「異性」「男女」を当たり前にしない。「相手」「ふたり」）。
- **詩的トーン**（天気のメタファーを軸に、断定より寄り添い）。
- **侮辱的に読まれうる表現を避ける**。
- 「結婚」「成婚」をゴールとして煽らない。「**好きな人**」「**ふたり**」「**過ごす日々**」を中核に。

## 3. 婚活ワードの線引き（重要・場所で違う）
- **bio・プロフィール文**：**二層構造**（2026-07-02 ふうか決裁で改定）。1行目に機能記述として「結婚相談所」使用可（例：「結婚相談所を、カウンセラーの口コミで選べるサイト」）。2行目以降は世界観の言葉（「好きな人」「ふたり」）で書く。ブランドコピーは従来どおり婚活語なし。
- **投稿本文・ハッシュタグ・コラム本文**：婚活/結婚相談所を**使ってOK**（SEO上むしろ必須）。ただし主役にしない。

## 4. 絵文字・著者・ビジュアル
- **絵文字**：サイト本文・記事は**なし**。**SNSキャプションは最小限・区切り用途で許容**。
- **著者**：実名「**ふうか**」を使う（E-E-A-T）。
- **顔出しはしない**。著者アイコン／登場ビジュアルは「**クレイで作ったふうか像**」。
- **声（ふうかの声）は利用可**：note 音声記事、リールのナレーション等（顔は出さない）＝顔なしの強い武器。
- フロントのビジュアル定義：ミニチュアクレイ風／背景 `#F5EEE6`（ウォームベージュ）／アクセント `#D4A090`（テラコッタ）。

- **創業ストーリー**：録音→Whisper ではなく、**Claude との対話式**で素材を引き出して文章化 → SNSキット **Part C** の構成で **note 1本目**に。営業デッキ／`/about/founder` と素材共有。

## 7. Story / belonging の禁止事項（ブランドの核を守る）
- **Story は編集ゲート制を維持＝公開投稿型（オープン UGC）にしない**（やらせ排除＝信頼の核）。
- Story の素材は**項目別の掲載同意（Web/SNS二次利用/写真/実名・仮名/相談所名/相手配慮）が無ければ下書きも作らない**。既定は仮名・年代ぼかし・クレイ情景のみ。
- 「ひとりじゃない（belonging）」は Story ではなく **Kinda Note 側の任意・匿名共有**で将来実現（Phase 3）。SNSでも「他人の物語の無断利用」はしない。

## 8. いま保留／依存（ふうかさん側）
- **アカウント確保**：note / Instagram / X のハンドル・bio（bio は婚活語なし）。
- **共通アイコン**：クレイふうか像を選定中 → 確定したら3媒体共通に（サイトの favicon は暫定で日の出マークに変更済み。最終アイコン確定後に合わせて差し替え可）。
- **問い合わせ返信の名義**：将来 hello@kinda.jp 名義で返信できるよう Google Workspace を検討中（SNSとは別レイヤー）。

## 9. Claude.ai ↔ Claude Code の役割分担
- **Claude.ai（SNS発信）**：投稿企画・文面・note記事・キャプション・カレンダー運用・創業ストーリー対話。
- **Claude Code（実装）**：記事インフラ（voices/story の MDX スキーマ・Article/Person/FAQPage JSON-LD・動的OGP・stage→クレイ画像・内部リンク枠）。記事を本サイトに載せる時はこちらへ。
