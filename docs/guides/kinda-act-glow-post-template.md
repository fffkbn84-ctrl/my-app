# Kinda act / glow 店舗掲載の型（アフィリエイト対応・2026-07-06 制定）

> バリューコマース審査通過（2026-07-06）を受けた、店舗掲載の標準手順。
> 原則：**提携の有無に関わらず「良い店」を先に選び、リンクは後から張る**（ユーザーファースト＝CLAUDE.md §8）。

## 1. 提携する広告プログラム（バリューコマース内）

| 優先 | プログラム | 用途 | 報酬目安 |
|---|---|---|---|
| 1 | **一休.comレストラン** | ホテルラウンジ・記念日レストラン（お見合い・顔合わせの定番帯） | 予約金額×1.1% |
| 2 | **ホットペッパーグルメ** | カフェ・カジュアルな店（1回目デート帯） | 来店110円/人 |
| 保留 | 食べログ・ぐるなび | 上記2つでほぼカバーできるため当面不要 | — |
| 検討 | OZmall（別ASP圏） | アフタヌーンティー・女性向けが強い。glow拡張時に再検討 | 来店300円 |

- VC管理画面 →「広告主を探す」で上記2つに**提携申請**（即時〜数日）。
- リンク生成は **MyLink（どこでもリンク）** で「その店舗の一休/HPGページURL」を指定して発行するのが基本。

## 2. 掲載する店の選定基準（順番を守る）

1. お見合い・初回デートに**本当に向いているか**（静かさ・席間隔・2時間居られるか・アクセス）
2. 掲載済みコラム（omiai-cafe-tokyo 等）や取材で根拠が語れるか
3. **最後に**アフィリエイト提携先（一休/HPG）に掲載があるか確認 → あればリンク化、なければ公式サイトURL

## 4. アフィリエイトの仕組み（実装済み・2026-07-06）

- `booking_url` にバリューコマース等のASPドメイン経由URLを入れると、店舗詳細ページ（/places/[id]）が**自動で判定**して：
  - 予約ボタンに `rel="sponsored"` が付く（SEO対応）
  - ボタン直下に「**PR：予約リンクは提携プログラム（アフィリエイト）を利用しています。掲載店の選定には影響しません。**」を表示（景表法・ステマ規制対応）
  - モバイル固定ボタンにも小さく「PR」が付く
- つまり**リンクを貼り替えるだけで表記が追従する**。手動でPRバッジを付ける作業は不要。
- 生URL（一休/HPGの通常URL）の間はPR表記は出ない（アフィリエイトではないため正しい挙動）。

## 5. 掲載手順

1. この型の §3 を埋める（コラム・公式サイト・一休/HPGページから収集）
2. admin（/admin/shops/new）または Claude Code 経由で shops に登録（badge=掲載店）
3. VC管理画面で MyLink を発行 → `booking_url` を差し替え
4. 本番で表示確認（PR表記・予約ボタン・地図）

## 6. SNS導線の型（将来運用）

- **X（B柱扱い）**：「結婚相談所のお見合い場所、どう選ぶ？」等の顕在ワード＋意見型 → リプに①該当コラム ②Kinda act店舗ページ の2リンク（W27勝ち型の応用）
- **IG（保存カルーセル）**：「お見合いで使いやすい店の条件5つ」型 → 最終スライドで act への保存誘導。キャプションにアフィリエイトリンクは**貼らない**（IGは外部リンク不可・PR表記管理が煩雑になるため、誘導先は kinda.jp のみ）
- SNS→kinda.jp店舗ページ→予約、の二段導線に統一する（PR表記をサイト側で一元管理できる）
