## このドキュメントの位置づけ

直近のユーザーレビューで判明した「初見ユーザーが迷子になる」問題を解決するための、**トップページ再構成 + サイト全体のナビゲーション統一**の指示書。

レビュアーから受けた指摘は次の3つ：

1. **初見でガイドがないと分かりにくい**（トップページの導線が曖昧）
2. **下層ページで現在地と戻り方が分からない**（パンくず・サブヘッダー不在）
3. **「カウンセラーを見たい」時にどこを触ればいいか分からない**（入口が散らばっている）

これらを解決しつつ、**SEO効果はむしろ高める**方向で改修する。Kinda の世界観（ミニチュアクレイ風・ウォームベージュ・「ふたりへ」）は維持する。

---

## 設計の最上位原則（判断に迷ったらここに戻る）

1. **Kinda note を主役にする**：Kinda は「翻訳ツール」であり、Kinda note こそがその核。トップページの主CTA は Kinda note への導線とする。Kinda type は副次的な選択肢として並列配置する。
2. **Kinda note は全ユーザー向け**：「まだ何も決まっていない人用」ではない。入会前〜交際後まで、ネガティブ・ポジティブ両方の感情で何度でも使える機能として位置づける。
3. **既存のデザイントークン（`C` オブジェクト）を維持する**。新規カラートークンは追加しない。
4. **既存コンポーネントを再利用する**。新規コンポーネントは必要最小限。
5. **画像は後から差し替え可能なプレースホルダー**で実装する（クリーム色矩形 + 中央に `[image: description]`）。
6. **絵文字は使わない**。すべて SVG アイコン化する。

### このセクションの目的

- 初見ユーザーへのガイドの代替（「自分はこれを使うべきか」を即座に判断できる）

  │ ・お見合いの後、ことばにならない違和感があった │

  │ ・複数の人で、気持ちが揺れている        │
  └──────────────────────────────┘


  気持ちが動いたとき

  ┌──────────────────────────────┐
  │ ・好きな人ができた、その気持ちを整理したい   │
  │ ・「好き」をどう伝えればいいか考えたい     │

### スタイル仕様

- **見出し「Kinda note は、こんな時に使えます」**：
  - Shippori Mincho、22px、`C.text`、中央揃え
  - 上下に区切り線（既存セクションと統一感を持たせる）
- **小見出し「ふと立ち止まったとき」「気持ちが動いたとき」**：
  - Shippori Mincho、16px、`C.textMid`、左揃え
  - 各ボックスの上 16px のマージン
- **ユースケースリスト（ボックス内）**：
  - 背景：`C.cardLight`（既存トークン、淡いクリーム）
  - 角丸：16px
  - パディング：20px
  - 行間：1.9
  - フォント：Hiragino Kaku Gothic ProN、14px、`C.text`
  - 各項目の頭：「・」（中点）、`C.textMid`
  - 項目間：12px の縦間隔
- **2つのボックス間の間隔**：24px
- **締めの一文「入会前から交際後まで、何度でも。」**：
  - Georgia serif、italic、16px、`C.text`、中央揃え
  - 上 32px、下 16px のマージン
- **CTAボタン**：
  - テキスト：「気持ちを整理する →」
  - スタイル：A セクションの主CTA より小さめ（パディング 14px × 36px）、`C.accent` 塗り、白文字
  - 遷移先：`/kinda-note`

### Claude Code への注意事項

- ユースケース文言を**勝手に変えない**。これは SEO キーワードと UX の両方を意識した文言で、特に「節目のとき、いまの自分を残しておきたい」は将来のマイページ天気ダッシュボード（フェーズ5）への伏線
- 「ふと立ち止まったとき」と「気持ちが動いたとき」という小見出しは、ネガポジを直接的に書かないための意図的な選択。「悩み」「うれしいこと」みたいに直接的に書かないこと
- ボックス内項目数は 4 で固定（増やさない・減らさない）

### SEO への配慮

ボックス内の各項目は、ロングテールキーワードと意味的に重なる：

| ユースケース | 対応キーワード |
|---|---|
| カウンセラーになんて伝えればいいか分からない | 「結婚相談所 カウンセラー 相談 仕方」 |
| お見合いの後、ことばにならない違和感 | 「お見合い 違和感」「お見合い 微妙」 |
| 交際中、なぜか不安が消えない | 「交際 不安」「結婚相談所 交際 続かない」 |
| 複数の人で、気持ちが揺れている | 「お見合い 複数 迷う」 |
| 好きな人ができた、その気持ちを整理 | 「好き 気持ち 整理」 |
| 「好き」をどう伝えればいいか | 「好きを伝える 方法」 |
| 大事なデートの前 | 「デート 前 緊張」「お見合い 前 不安」 |
| 節目のとき、いまの自分を残しておきたい | 「自分の気持ち 記録」 |

これらのテキストは**そのまま検索エンジンに読まれる**ので、文言を変える時は SEO への影響も考慮すること。

# 改修パート2：全下層ページのナビゲーション統一

## 2-1. パンくずリストの全下層ページへの追加

### 設置場所

各下層ページの**コンテンツの最上部**（ヘッダー固定領域の直下、メインコンテンツの一番上）。

### スタイル仕様

```tsx
<nav aria-label="パンくずリスト" className="breadcrumb">
  <ol>
    <li><a href="/">ホーム</a></li>
    <li aria-hidden="true">›</li>
    <li><a href="/type">Kinda type</a></li>
    <li aria-hidden="true">›</li>
    <li aria-current="page">診断結果</li>
  </ol>
</nav>
```

- フォント：12px、`C.textMid`
- 区切り文字：`›`（U+203A、SINGLE RIGHT-POINTING ANGLE QUOTATION MARK）、左右に 4px のマージン
- リンク部分：`C.textMid`、ホバー時 `C.accent`、下線なし
- 現在ページ（最後の項目）：`C.text`、リンクではなく `<span>` または `<li aria-current="page">`
- 上下マージン：上 12px、下 16px、左右 16px
- ホーム以外の最初の項目は機能のセクショントップ（`/type` `/talk` 等）

### 構造化データ（schema.org BreadcrumbList）

SEO 効果を最大化するため、`<script type="application/ld+json">` で BreadcrumbList を埋め込む：

```html
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "BreadcrumbList",
  "itemListElement": [
    {
      "@type": "ListItem",
      "position": 1,
      "name": "ホーム",
      "item": "https://kinda.jp/"
    },
    {
      "@type": "ListItem",
      "position": 2,
      "name": "Kinda type",
      "item": "https://kinda.jp/type"
    },
    {
      "@type": "ListItem",
      "position": 3,
      "name": "診断結果"
    }
  ]
}
</script>
```

### 各セクションのパンくず例

| 現在ページ | パンくず |
|---|---|
| `/type` | ホーム › Kinda type |
| `/type/quiz` | ホーム › Kinda type › 診断中 |
| `/type/result/[id]` | ホーム › Kinda type › 診断結果 |
| `/talk` | ホーム › Kinda talk |
| `/talk/counselors/[id]` | ホーム › Kinda talk › （カウンセラー名） |
| `/kinda-note` | ホーム › Kinda note |
| `/kinda-note/result/[id]` | ホーム › Kinda note › 結果 |
| `/act` | ホーム › Kinda act |
| `/glow` | ホーム › Kinda glow |
| `/story` | ホーム › ふたりの物語 |
| `/story/[id]` | ホーム › ふたりの物語 › （タイトル抜粋） |
| `/about` | ホーム › Kindaについて |

### 実装パターン

共通コンポーネント `<Breadcrumb items={...} />` を作成し、各下層ページの `page.tsx` で配列を渡す形にする：

```tsx
// app/_components/Breadcrumb.tsx
type BreadcrumbItem = { label: string; href?: string };

export function Breadcrumb({ items }: { items: BreadcrumbItem[] }) {
  return (
    <>
      <nav aria-label="パンくずリスト" className="breadcrumb">
        <ol>
          {items.map((item, i) => (
            <React.Fragment key={i}>
              {i > 0 && <li aria-hidden="true">›</li>}
              <li {...(i === items.length - 1 ? { 'aria-current': 'page' } : {})}>
                {item.href ? <a href={item.href}>{item.label}</a> : item.label}
              </li>
            </React.Fragment>
          ))}
        </ol>
      </nav>
      <BreadcrumbStructuredData items={items} />
    </>
  );
}
```

`BreadcrumbStructuredData` コンポーネントで JSON-LD を出力する。

## 2-2. サブヘッダーの導入

### スタイル仕様
