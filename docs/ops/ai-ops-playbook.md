### 最初に着手した3つ（本日実装済み）と理由

1. **#1 /sns-pack** — 申告された最大ボトルネック「SNS配信などの集客コンテンツの作成」に直撃。毎週使う。
2. **#4 content-qa** — すべてのコンテンツの公開ゲートになり、#1〜#3の出力品質を機械的に担保。ステマ規制・同意漏れという「事故ると一番痛い」リスクを塞ぐ。
3. **#2+#3 story/column Skill** — 毎週の定型投稿。仕様が既に固まっており（CLAUDE.md §5・既存記事）、Skill化の費用対効果が最大。

## 4. 明日からの運用手順

1. **朝（5分）**: Notionカレンダーの当日分をコピーして投稿（リンクはセルフリプ）。リプ返し。
2. **週1回（30分）**: Claude Codeで `/sns-pack`（ソース=直近コラム or 今週のテーマ）→ 生成物を確認・微修正 → Notion投入を承認 → 翌週分の下書きが揃う。
3. **コンテンツ投稿時**: `/kinda-column` または `/kinda-story` → `npm run qa:content` が error 0 → PR → マージ後に Vercel production が READY か確認。
4. **月1回**: X/IG/noteの数字をメモして Claude に渡し「伸びた型・翌月方針」を分析させる（→ #8 を作ると半自動化）。
